GitHub 는 웹 기반의 프로젝트 관리(버전 관리) 서비스로, 협업 개발에 최적화되어 있으
며 저장소(Repository)를 제공하여 전 세계 오픈 소스 S/W의 총집합체 역할
git init : 선언(박스 가져오기)
git add: 깃 상자에 폴더 내용을 추가(박스 안에 넣기)
git commit : 깃 상자자에 있는 모든 파일을 깃 버전으로 저장(택배 포장)
git push : 저장된 버전을 깃허브 서버에 업로드

프론트엔드: 상호작용,몰입감(반응형 디자인, 마이크로 인터랙션)이 중요
백엔드: 서버 중앙 집중형 -> 분산형(블록체인)으로 변함.

부트스트랩 5:  강력하고 기능이 풍부한 프론트엔드 툴킷

CDN(Content Delivery Network) 방식 : 부트스트랩 파일을 다운로드하지 않고, 서버에서 바로 가져와 사용하는 방식(웹사이트 로딩 속도 빨라지고, 항상 최신 버전의 부트스트랩 사용 가능)

<meta> 태그: 문자 인코딩, 뷰포트 설정 등 페이지의 전반적인 정보
<link> 태그: 외부 CSS파일이나 외부 자원을 연결할 때 사용
<script> 태그: 자바스크립트 파일 연결할 때 사용

하이퍼링크(hyperlink): <a href=" "> 태그를 사용하여 페이지 연결을 지정,
href 속성으로 링크 주소(url)를 가짐.

잘못된 주소(예: 404 - Resource Not Found )를 넣으면 웹 서버는
index.html 을 로딩하지 못하고 HTTP 상태 코드 404 를 반환

링크 대상 속성: _self (현재 탭), _blank (새 탭), _parent (부모 프레임), _top (최상위
프레임) 4가지가 있음.

이미지 로딩: <img src=" "> 태그를 사용하여 이미지를 나열, 외부 주소로부
터 이미지를 로딩

상대 경로 ( ../ , ./ ): 현재 파일의 위치를 기준으로 이미지 파
일의 위치를 알려주는 거
절대 경로 ( / , http://... ): 웹사이트의 가장 최상위 루트 폴
더나 특정 웹 주소부터 시작하는 경로

CSS는 기본(브라우저 기본 스타일) -> 외부(링크된 CSS 파일) -> 내부/인라인

그리드 : 웹페이지를 가상의 격자무늬로 나누어 요소를 배치하는 방식
row g-4 그리드 활용: 현재 그리드는 row g-4 를 사용하며, g-4 는 컬럼 사이의
간격(gutter)을 4단계로 조절하는 클래스

자바스크립트는 웹 페이지의 연결 과 동작 을 담당하는 언어
 HTML이 웹 페이지의 구조를, CSS가 뷰(스타일) 를 담당한다면, 자바스크립트는 버튼 클릭 시 특정 동작
(예: 불 켜기)이 일어나게 하는 등 상호작용 을 구현
웹, 모바일, 백엔드 등 전 분야 에서 활용되며 범용성과 호환성 이 매우 높음.
인터프리터 언어 이자 (함수 + 객체) 지향 언어 의 특징을 가지며, 웹 브라우저에 내장된 JS 전용 엔진
(예: 구글 크롬 V8 엔진)을 통해 실행

호이스팅 은 자바스크립트의 오래된 특징 중 하나로, 코드가 실제로 실행되기 전에 변수
및 함수 선언부가 해당 스코프의 상단으로 끌어올려지는 현상

div : 그저 공간만 나누는 역할
Section : 여기는 검색 결과 섹션이라는 의미를 부여해서 나눔, 웹 접근성이나 검색 엔진 최적화에 더 좋음

검색어 입력
: 검색어 정리 -> 데이터 필터링 -> 결과 표시 -> HTML 동적 생성 -> 화면 전환
